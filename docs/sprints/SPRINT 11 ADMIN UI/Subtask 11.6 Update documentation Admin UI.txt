3. docs/api-endpoints.md
   - Add section "## Admin Hooks"
````markdown
   ## Admin Hooks
   
   ### useAdminStats
   **File:** `src/hooks/admin/useAdminStats.ts`
   
   Fetches dashboard statistics.
   
   **Returns:**
   - `stats` - Statistics object
   - `loading` - Loading state
   - `error` - Error message
   - `refetch` - Refresh function
   
   **Backend RPC:** `get_admin_stats`
   
   ### usePendingReports
   **File:** `src/hooks/admin/usePendingReports.ts`
   
   Fetches list of pending reports.
   
   **Backend RPC:** `list_pending_reports`
   
   ### useReportDetails
   **File:** `src/hooks/admin/useReportDetails.ts`
   
   Fetches complete report details with context.
   
   **Backend RPC:** `get_report_details_with_context`
   
   ### useResolveReport
   **File:** `src/hooks/admin/useResolveReport.ts`
   
   Resolves a report with action.
   
   **Actions:**
   - `dismiss` - Dismiss report without action
   - `remove_content` - Remove the reported content
   - `suspend_user` - Suspend the user
   
   **Backend RPC:** `resolve_report`
   
   ### useUserSearch
   **File:** `src/hooks/admin/useUserSearch.ts`
   
   Searches users with filters.
   
   **Parameters:**
   - `query: string` - Search query
   - `status: 'all' | 'active' | 'suspended'` - Status filter
   
   **Backend RPC:** `search_users_admin`
   
   ### useSuspendUser
   **File:** `src/hooks/admin/useSuspendUser.ts`
   
   Suspends or unsuspends users.
   
   **Functions:**
   - `suspendUser(userId, reason)`
   - `unsuspendUser(userId, reason)`
   
   **Backend RPCs:** `suspend_user_with_reason`, `unsuspend_user`
   
   ### useAuditLog
   **File:** `src/hooks/admin/useAuditLog.ts`
   
   Fetches audit log with filtering.
   
   **Parameters:**
   - `actionType: string` - Filter by action type
   
   **Returns:** Paginated audit log entries
````
